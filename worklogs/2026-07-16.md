# 2026-07-16

## 完成進度

- 依確認畫面精簡 Excel-like 篩選，只保留搜尋、全選、唯一值清單與套用／取消／清除；移除選單排序、頁籤與條件篩選。
- 依使用者指示執行正式 build，更新 FabUI 主 bundle、Vue 2 wrapper 與 jQuery wrapper。
- 新增 `allowFiltering` 共用欄位篩選開關與 `setAllowFiltering(value)`；關閉時清除並隱藏 Search Row／Excel-like 篩選，右下角 Quick Search 保持可用。
- Grid popup 觸發位置由左上角列頭 cell 改為欄位 Header Row 右鍵；Search Row 與資料列維持原行為。
- `grid.js` 工具列原「搜尋列」checkbox 改為「篩選」，控制 `allowFiltering`；`showSearchRow` 改由 Header 右鍵 popup 選擇模式並獨立保存。
- Header 篩選 icon 的 hit area 從圖示下方延伸到 Header title row 底部，圖示尺寸與位置維持不變，降低點擊難度。
- Excel-like 篩選 popup 支援 `Escape` 關閉；未按「套用」的值選取草稿不會變更目前篩選。
- 在 AGENTS 加入通用 popup 規範：所有 popup 均須支援 `Escape` 關閉，且不得因此提交或清除未確認內容。
- 修正凍結欄小數造成 layout crash、零／無效列高造成 virtual range `NaN`、事件派送期間解除 handler 跳過後續 handler，以及 Excel-like 唯一值超過上限時誤排除未列出值。
- 捲動仍位於既有 overscan range 時不再安排完整 DOM 重繪；scroll range 跨界時沿用既有 layout，並消除 refresh／render 間的重複 layout 計算。
- `observeItemsSource` 將同一同步批次的 Proxy mutation 合併為一次 `applyView()` 與 `refresh()`；1,000 筆陣列的 `splice(0, 500)` 由 1,001 次更新降為 1 次。
- 將 `fabgrid.js` 依完整領域拆為 `fabgrid-view.js`、`fabgrid-filter-ui.js`、`fabgrid-selection.js`、`fabgrid-editor-runtime.js`，主檔保留 factory、公共 API、事件與模組協調。
- 將 `alternatingRows` 改為 `alternatingRowStep`，預設 `1`；`false` 關閉交替色，正整數控制每幾列切換一次背景，並同步 core、Vue wrapper、Demo、API 文件與 smoke 契約。
- 正式定義 `formatItem.addHandler((g, e) => {})` Wijmo-compatible 契約，新增 `fabui.CellType`、GridPanel `getCellData()`、`g.rows[e.row].dataItem` 與集中式事件參數建立流程，涵蓋 Header、Footer、資料 cell、列頭及角落 cell。
- 新增 `getJson()`、`exportJson()`、`importJson()` API；預設完整 round-trip `itemsSource`，可選擇匯出目前 view，匯入支援 JSON 字串、Array、envelope object 與 Blob／File。
- 新增 `fabui.Control.getControl(elementOrSelector)` 與 `grid.hostElement`；FabGrid 建立後自動登記 host，`dispose()` 時自動解除，可直接用 `'#' + grdId` 取得 Grid instance。
- 新增 Wijmo-compatible `Row`／`GroupRow` class hierarchy；`g.rows` 與 `selectedRows` 改回傳 instance，群組 header／footer 使用 `GroupRow`，只由 `fabui.FabGrid.Row`／`fabui.FabGrid.GroupRow` 公開。
- 調整 Filterable Header 文字維持垂直置中、漏斗 icon 疊在右上方；擴大透明 hit area，並讓 resize handle 層級高於 icon，確保 filter 不觸發排序且欄寬拖曳／雙擊 AutoFit 正常。
- 為 Filterable Header 右側預留漏斗 icon 空間；欄寬足夠時，靠右對齊的數字標題不再與 filter icon 重疊。
- Filterable Header 改為自適應 icon 空間：欄寬足夠時維持間距，欄寬不足時讓標題延伸到 filter icon 下方，避免「文字日期」、「應付金額」過早截斷。
- Excel-like filter popup 依值清單內容、Grid 可視範圍與瀏覽器 viewport 動態計算高度；搜尋與按鈕列固定，值清單填滿剩餘空間並獨立捲動，避免 popup 超出螢幕。
- Excel-like filter popup 高度計算額外保留 12px 底部陰影空間，避免 Grid 的 `overflow: hidden` 裁掉下方陰影。
- Excel-like filter popup 水平定位優先與所屬 Header Cell 左側切齊；右側空間不足時才向左位移，並維持 Grid／viewport 邊界限制。
- 新增 `selectionMode: 'CellRange'`，支援滑鼠拖曳、`Shift + Click`、`Shift + 方向鍵`、`selectRange()`、範圍樣式與 TSV clipboard copy；GroupRow／group footer 不加入範圍資料。
- 新增 `highlightActiveRow`，預設為 `true` 以維持既有外觀；設為 `false` 時只隱藏 active row 背景，不影響 active cell、CellRange 或多選列。
- 修正 `CellRange` 的 active cell 與 range 外框疊加造成邊框看似多 1px；active cell 所在邊界改回一般格線，讓 `activeCellBorder` 維持設定粗細。
- 新增 Wijmo-compatible Column `cellTemplate`，支援 function／string template、`ctx + cell` callback、runtime assignment 自動 invalidate，並維持 editor、clipboard 與 export 使用原始 binding value。
- 調整 function `cellTemplate` 的 style 契約：callback 前後自動保存、還原與合併 inline style；`cell.style = customStyle` 只疊加視覺樣式並保護 Grid 定位尺寸，`cell.style = null` 還原原 Grid style。
- 新增 Wijmo-compatible `Control.addEventListener()`／`removeEventListener()` 與 `FabGrid.hitTest()`；managed DOM listener 會在 `dispose()` 自動解除，Search Row 以 `ColumnHeader` cellType 與 `isSearchRow` 辨識。
- 左上角欄位選擇器 popup 新增 `Escape` 關閉，並補強 `pointerdown`／`mousedown` 點擊 popup 外部關閉；popup 內部與觸發按鈕維持開啟。
- 修正資料格第一次點擊重繪 DOM 後無法產生 `dblclick`；FabGrid 現在會在必要時對重繪後的同一資料格補發一次相容事件，讓 `addEventListener(hostElement, 'dblclick', ...)` 與 `hitTest(e)` 正常取得 Data Cell。
- 修正 `selectionMode: 'CellRange'` 在 `pointerdown` 即重繪資料格而遺失雙擊事件；改由同一資料格的連續 pointer 操作辨識雙擊，並在未拖曳時補發相容 `dblclick`。
- 補強 CellRange pointer 契約：雙擊必須落在同一資料格、`pointercancel` 不觸發雙擊、完成 pointer 選取後不再由 click 重複 render；欄位拖曳、欄寬調整、範圍拖曳、捲軸拖曳與資料列拖曳的 document pointer listener 改為互動期間才綁定，並修正 source hit-test 只引用 `src` 樣式。
- 同步更新 `AGENTS.md` 與 `README.md` 的 CellRange／document pointer listener 契約；新增回歸測試後 `npm test` 108 項全數通過，source-mode 真實雙擊與 2×2 拖曳選取通過，正式 build 與 `node build/smoke.cjs` 驗證完成。
